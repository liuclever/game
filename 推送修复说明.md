# 推送修复后的提交到远程

## 问题原因

**不是终端编码问题**，而是：
- ✅ 本地提交已修复（c42766e）
- ❌ 远程仓库仍指向乱码提交（41d03fa）
- IDE/客户端显示的是远程分支的乱码提交

## 解决方案

需要将修复后的提交推送到远程仓库：

```bash
# 强制推送（会覆盖远程的乱码提交）
git push --force-with-lease origin feature/social-mall
```

### 为什么使用 --force-with-lease？

`--force-with-lease` 比 `--force` 更安全：
- 如果远程有其他人的新提交，会拒绝推送
- 防止意外覆盖其他人的工作

### 如果推送被拒绝

如果提示远程有更新，先拉取：
```bash
git fetch origin
git rebase origin/feature/social-mall
git push --force-with-lease origin feature/social-mall
```

## 推送后的效果

推送后：
- ✅ 远程分支也会显示正确的提交消息
- ✅ IDE/客户端刷新后会显示正确的中文
- ✅ 所有团队成员看到的都是修复后的提交

## 注意事项

⚠️ **如果这是共享分支**：
1. 通知团队成员即将强制推送
2. 确保没有其他人基于旧提交工作
3. 推送后，团队成员需要：
   ```bash
   git fetch origin
   git reset --hard origin/feature/social-mall
   ```

## 验证

推送后验证：
```bash
# 查看远程提交
git log origin/feature/social-mall --oneline -1
# 应该显示：c42766e 更新项目代码到feature/social-mall分支
```
