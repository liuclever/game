# 本地调试指南

## 📁 项目结构
这是一个修仙题材的游戏项目，采用 Flask 后端 + Vue 前端架构。

## 🔧 环境准备

### 必需软件
- Python 3.10+
- Node.js 18+
- MySQL 8.0

### 检查环境
```bash
python --version
node --version
npm --version
mysql --version
```

---

## 🗄️ 数据库配置

### 1. 创建数据库
```sql
CREATE DATABASE game_tower CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 导入数据库结构
```bash
mysql -u root -p game_tower < game_tower.sql
```

### 3. 修改数据库连接配置
编辑文件：`infrastructure/db/connection.py`

```python
DB_CONFIG = {
    'host': 'localhost',
    'port': 3306,
    'user': 'root',
    'password': '你的MySQL密码',  # ⚠️ 改成你本地的密码
    'database': 'game_tower',
    'charset': 'utf8mb4',
    'cursorclass': DictCursor,
}
```

## 🚀 启动项目

### 方式一：使用批处理脚本（推荐）

**1. 首次启动（安装依赖）：**
```bash
dev_helper\首次安装依赖.bat
```

**2. 日常启动：**
- 双击 `dev_helper\启动后端.bat` 启动后端
- 双击 `dev_helper\启动前端.bat` 启动前端

### 方式二：手动启动

**终端 1 - 后端：**
```bash
python -m interfaces.web_api.app
```

**终端 2 - 前端：**
```bash
cd interfaces\client
npm run dev
```

## 🐛 调试技巧

### 1. 查看后端日志
后端运行的终端会实时显示请求日志和错误信息

### 2. 查看前端控制台
浏览器按 F12 打开开发者工具，查看 Console 和 Network 标签

### 3. 数据库调试
```bash
# 连接数据库
mysql -u root -p game_tower

# 查看玩家数据
SELECT * FROM players LIMIT 10;

# 查看幻兽数据
SELECT * FROM player_beasts LIMIT 10;
```

### 4. 修改游戏配置
游戏数值配置在 `configs/` 目录下，修改 JSON 文件后重启后端即可生效

## 🔍 项目架构说明

```
├── domain/              # 核心游戏规则（战斗、升级、掉落逻辑）
├── application/         # 业务服务（签到、战斗、地图等操作）
├── interfaces/          # 前端界面和 API 路由
│   ├── web_api/        # Flask 后端 API
│   └── client/         # Vue 前端项目
├── infrastructure/      # 数据库、配置加载等技术实现
├── configs/            # 游戏数值配置（JSON 文件）
└── tests/              # 测试用例
```

---

## 💡 开发新功能流程

1. **定义规则** - 在 `domain/` 编写核心逻辑
2. **实现服务** - 在 `application/services/` 实现业务流程
3. **添加路由** - 在 `interfaces/web_api/routes/` 添加 API
4. **配置数值** - 在 `configs/` 添加配置文件
5. **编写测试** - 在 `tests/` 添加测试用例

## ❓ 常见问题

### Q: 后端启动失败，提示数据库连接错误
A: 检查 `infrastructure/db/connection.py` 中的数据库密码是否正确

### Q: 前端启动失败，提示端口被占用
A: 修改 `interfaces/client/vite.config.js` 中的端口号

### Q: 修改代码后没有生效
A: 前端会自动热更新，后端需要重启服务

### Q: 如何创建测试账号
A: 直接在数据库中插入数据，或通过前端注册功能
