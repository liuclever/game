# 召唤之王挑战赛需求文档

## 功能概述

召唤之王挑战赛是一个跨服竞技系统，分为预选赛和正赛两个阶段。

## 赛区系统

### 赛区分配
- 全服玩家分为1区和2区
- 建号时随机分配赛区
- 保证两个赛区人数基本一致（差距不超过1人）

### 实现方式
```python
# 在注册时分配赛区
area_1_count = count(area_index=1)
area_2_count = count(area_index=2)
area_index = 1 if area_1_count <= area_2_count else 2
```

## 预选赛阶段

### 时间安排
- **报名时间**：每周一 00:00 - 23:59
- **比赛时间**：每周二 00:00 - 周四 23:59

### 报名规则
- 星期一可以报名
- 报名后随机分配赛区内的初始排名
- 例如：赛区有100人，新报名的玩家随机分配到第50-100名之间

### 挑战规则
1. **每日挑战次数**：15次免费挑战
2. **匹配规则**：
   - 只能挑战排名比自己高的玩家
   - 显示前5名可挑战的玩家
   - 例如：我是第12名，显示第7-11名
   - 如果不足5人，显示实际人数
3. **挑战冷却**：
   - 第一次挑战无冷却
   - 第二次及以后挑战需要等待60秒
4. **并发控制**：
   - 使用数据库锁防止同时挑战同一个人
   - 使用 `SELECT ... FOR UPDATE` 锁定排名记录

### 战斗结果
- **胜利**：交换排名，连胜+1，获得20000铜钱
- **失败**：排名不变，连胜清零，获得2000铜钱

## 正赛阶段

### 晋级规则
- **时间**：每周四 23:59 预选赛结束
- **晋级条件**：每个赛区前16名进入正赛
- **总人数**：32人（两个赛区各16人）

### 赛程安排
1. **32强赛**：周五 12:00
   - 32人随机匹配
   - 如果不足32人，部分选手轮空直接晋级
   - 胜者进入16强
   
2. **16强赛**：周五 13:00
   - 16人随机匹配
   - 胜者进入8强
   
3. **8强赛**：周五 14:00
   - 8人随机匹配
   - 胜者进入4强
   
4. **4强赛**：周五 15:00
   - 4人随机匹配
   - 胜者进入2强
   
5. **决赛**：周五 16:00
   - 2人对决
   - 胜者成为冠军

### 轮空处理
- 如果参赛人数不是2的幂次方，部分选手轮空
- 轮空选手直接晋级下一轮
- 轮空选手也能获得当前轮次的奖励

## 数据库表设计

### king_challenge_rank（排名表）
```sql
CREATE TABLE king_challenge_rank (
    user_id INT PRIMARY KEY,
    area_index INT NOT NULL DEFAULT 1,  -- 赛区编号
    rank_position INT NOT NULL,          -- 排名
    win_streak INT DEFAULT 0,            -- 连胜场数
    total_wins INT DEFAULT 0,            -- 总胜场
    total_losses INT DEFAULT 0,          -- 总负场
    today_challenges INT DEFAULT 0,      -- 今日挑战次数
    last_challenge_date DATE,            -- 最后挑战日期
    last_challenge_time DATETIME,        -- 最后挑战时间（用于冷却）
    is_registered TINYINT(1) DEFAULT 0,  -- 是否已报名本周
    INDEX idx_area_rank (area_index, rank_position)
);
```

### king_final_stage（正赛阶段表）
```sql
CREATE TABLE king_final_stage (
    id INT AUTO_INCREMENT PRIMARY KEY,
    season INT NOT NULL,                 -- 赛季编号
    user_id INT NOT NULL,                -- 玩家ID
    stage VARCHAR(20) NOT NULL,          -- 阶段：32/16/8/4/2/champion
    match_id INT,                        -- 对战ID
    opponent_id INT,                     -- 对手ID
    is_bye TINYINT(1) DEFAULT 0,        -- 是否轮空
    is_winner TINYINT(1),               -- 是否胜利
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_season_stage (season, stage),
    INDEX idx_user (user_id)
);
```

### king_reward_claimed（奖励领取记录）
```sql
CREATE TABLE king_reward_claimed (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    season INT NOT NULL,
    reward_tier VARCHAR(20),             -- 奖励档位
    claimed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_season (user_id, season)
);
```

## 并发控制

### 挑战锁
```python
# 使用数据库行锁
def challenge_with_lock(attacker_id, defender_id):
    with transaction():
        # 锁定双方的排名记录
        attacker = execute_query(
            "SELECT * FROM king_challenge_rank WHERE user_id = %s FOR UPDATE",
            (attacker_id,)
        )
        defender = execute_query(
            "SELECT * FROM king_challenge_rank WHERE user_id = %s FOR UPDATE",
            (defender_id,)
        )
        
        # 检查冷却时间
        if attacker['last_challenge_time']:
            cooldown = 60  # 60秒
            elapsed = (now - attacker['last_challenge_time']).total_seconds()
            if elapsed < cooldown:
                raise Exception(f"冷却中，还需等待{int(cooldown - elapsed)}秒")
        
        # 执行战斗
        result = do_battle(attacker_id, defender_id)
        
        # 更新排名
        if result.is_victory:
            swap_ranks(attacker_id, defender_id)
        
        # 更新挑战时间
        execute_update(
            "UPDATE king_challenge_rank SET last_challenge_time = NOW() WHERE user_id = %s",
            (attacker_id,)
        )
```

## 定时任务

### 每周一 00:00
- 开启报名
- 清空上周的正赛数据
- 重置所有玩家的报名状态

### 每周二 00:00
- 关闭报名
- 开始预选赛

### 每周四 23:59
- 结束预选赛
- 统计各赛区前16名
- 生成正赛对阵表

### 每周五 12:00/13:00/14:00/15:00/16:00
- 自动执行各轮次比赛
- 发放奖励
