## 数据库表结构

### king_challenge_rank（排名表）
```sql
CREATE TABLE king_challenge_rank (
    user_id INT PRIMARY KEY,
    area_index INT NOT NULL DEFAULT 1,           -- 赛区编号
    rank_position INT NOT NULL,                  -- 排名
    win_streak INT NOT NULL DEFAULT 0,           -- 连胜场次
    total_wins INT NOT NULL DEFAULT 0,           -- 总胜场
    total_losses INT NOT NULL DEFAULT 0,         -- 总负场
    today_challenges INT NOT NULL DEFAULT 0,     -- 今日挑战次数
    last_challenge_date DATE DEFAULT NULL,       -- 上次挑战日期
    last_challenge_time DATETIME DEFAULT NULL,   -- 最后挑战时间（冷却）
    is_registered TINYINT(1) NOT NULL DEFAULT 0, -- 是否已报名本周
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_area_rank (area_index, rank_position)
);
```

### king_final_stage（正赛阶段表）
```sql
CREATE TABLE king_final_stage (
    id INT AUTO_INCREMENT PRIMARY KEY,
    season INT NOT NULL,                         -- 赛季编号
    user_id INT NOT NULL,                        -- 玩家ID
    stage VARCHAR(20) NOT NULL,                  -- 阶段：32/16/8/4/2/champion
    match_id INT DEFAULT NULL,                   -- 对战ID
    opponent_id INT DEFAULT NULL,                -- 对手ID
    is_bye TINYINT(1) NOT NULL DEFAULT 0,       -- 是否轮空
    is_winner TINYINT(1) DEFAULT NULL,          -- 是否胜利
    battle_time DATETIME DEFAULT NULL,           -- 战斗时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_season_stage (season, stage),
    INDEX idx_user (user_id)
);
```

### king_season_config（赛季配置表）
```sql
CREATE TABLE king_season_config (
    season INT PRIMARY KEY,
    start_date DATE NOT NULL,                    -- 赛季开始日期
    end_date DATE NOT NULL,                      -- 赛季结束日期
    status VARCHAR(20) NOT NULL DEFAULT 'registration', -- 状态
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## API接口

### GET /api/king/info
获取挑战赛信息

**响应**：
```json
{
  "ok": true,
  "areaName": "1赛区",
  "totalPlayers": 100,
  "myRank": 12,
  "winStreak": 5,
  "todayChallenges": 3,
  "todayMax": 15,
  "challengers": [...],
  "cooldownRemaining": 45,
  "isRegistered": true
}
```

### POST /api/king/challenge
发起挑战

**请求**：
```json
{
  "targetUserId": 123
}
```

**响应**：
```json
{
  "ok": true,
  "win": true,
  "message": "恭喜！你击败了xxx，排名上升至第10名！获得20000铜钱",
  "newRank": 10,
  "reward": 20000
}
```

### POST /api/king/register

**响应**：
```json
{
  "ok": true,
  "message": "报名成功！预选赛将于周二开始"
}
```

## 注意事项

1. **赛区分配**：只在首次创建排名记录时分配，之后不会改变
2. **冷却时间**：60秒，从挑战完成时开始计算
3. **报名限制**：只能在星期一报名，其他时间不显示报名按钮
4. **并发控制**：使用行锁，如果两个人同时挑战同一个目标，后者会等待
5. **事务回滚**：任何错误都会回滚事务，保证数据一致性
