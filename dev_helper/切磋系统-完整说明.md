# 切磋系统完整说明

## 核心特点

### ✅ 不消耗资源
- **不消耗活力**（虽然战报显示-15，但实际不扣除）
- **不消耗捕捉球**
- **不消耗任何道具**

### ✅ 不影响排名
- 切磋结果不影响任何排行榜
- 纯粹的友谊赛

### ✅ 记录战绩
- 记录切磋次数和胜负
- 可以查看切磋战绩统计
- 显示胜率

### ✅ 真实战斗
- 使用完整的PVP战斗引擎
- 双方幻兽真实对战
- 支持所有技能系统

## 使用场景

### 1. 测试实力
- 与好友切磋，测试幻兽配置
- 不消耗资源，可以随意尝试

### 2. 练习战斗
- 熟悉战斗系统
- 测试不同的幻兽组合

### 3. 社交互动
- 与其他玩家互动
- 建立友谊

## 使用流程

### 方式一：从玩家主页
1. 点击任何玩家名字
2. 进入玩家详情页
3. 点击"战绩"旁边的"切磋"按钮
4. 瞬间显示切磋结果

### 方式二：从竞技主页
1. 访问竞技主页（`/arena/index`）
2. 查看连胜竞技场下方的对手列表
3. 点击"切磋"按钮
4. 跳转到连胜竞技场进行战斗

### 方式三：从好友列表
1. 访问好友列表页面
2. 点击好友名字进入主页
3. 点击"切磋"按钮

## 战报内容

### 显示信息
- 凝神香加成：无（切磋不受凝神香影响）
- 对战双方名字
- 幻兽列表（不显示经验，因为切磋不给经验）
- 战斗结果：完美胜利(⚬⚬⚬⚬:×××) / 胜利 / 失败
- 活力消耗：显示-15（实际不消耗）
- 战斗过程：每回合的攻击详情
- 切磋战绩：X胜/Y负（胜率Z%）

## 数据库表

### spar_records（切磋记录表）
```sql
CREATE TABLE spar_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    attacker_id INT NOT NULL COMMENT '发起方玩家ID',
    defender_id INT NOT NULL COMMENT '被挑战方玩家ID',
    is_victory TINYINT(1) NOT NULL DEFAULT 0 COMMENT '发起方是否胜利',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '切磋时间'
);
```

## API接口

### POST /api/player/spar
发起切磋

**请求：**
```json
{
  "target_id": 123
}
```

**响应：**
```json
{
  "ok": true,
  "win": true,
  "message": "切磋胜利！",
```

**请求参数：**

2. 用账号A登录

7. 查看切磋战绩统计

## 注意事项

1. **不消耗活力**：虽然战报显示-15，但实际不扣除
2. **不给经验**：幻兽不会获得经验
3. **不能自己切磋**：不能与自己的账号切磋
4. **需要幻兽**：双方都需要有出战幻兽
5. **真实战斗**：使用完整的战斗引擎，结果基于实力
