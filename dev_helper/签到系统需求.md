# 签到系统需求文档

## 功能概述

签到系统根据玩家等级发放铜钱奖励，连续签到5天及以上奖励翻倍。

## 核心规则

### 1. 颁发者名字
- 签到时显示颁发者的名字
- 颁发者从盟战排行榜前三的联盟中随机选一个
- 颁发者名字为联盟的名字

### 2. 连续签到奖励
- 连续签到5天及以上：初始铜钱×2
- 中途中断签到：回到初始铜钱数
- 需要记录连续签到天数

### 3. 初始铜钱数（按等级）

| 等级范围 | 初始铜钱 |
|---------|---------|
| 1-9级 | 5024 |
| 10-19级 | 10050 |
| 20-29级 | 15120 |
| 30-39级 | 20350 |
| 40-49级 | 25345 |
| 50-59级 | 30230 |
| 60-69级 | 35210 |
| 70-79级 | 40170 |
| 80-89级 | 45280 |
| 90-100级 | 50120 |

## 数据库设计

### player表需要的字段
```sql
last_signin_date DATE DEFAULT NULL,           -- 上次签到日期
consecutive_signin_days INT DEFAULT 0,        -- 连续签到天数
```

### alliance表（联盟表）
需要查询盟战排行榜前三的联盟

## 实现逻辑

### 签到流程
1. 检查今天是否已签到
2. 计算连续签到天数
   - 如果上次签到是昨天：连续天数+1
   - 否则：连续天数重置为1
3. 根据等级获取初始铜钱
4. 如果连续签到≥5天：铜钱×2
5. 从盟战排行榜前三联盟中随机选一个作为颁发者
6. 发放奖励
7. 更新签到记录

### 示例
```
玩家等级：25级
初始铜钱：15120
连续签到：6天
实际获得：15120 × 2 = 30240铜钱
颁发者：天下第一盟
```

