# 修复召唤之王挑战赛导入错误

## 🐛 问题描述

### 错误信息
```
module 'application.services' has no attribute 'player_repo'
挑战失败: module 'application.services' has no attribute 'player_repo'
```

### 触发场景
在召唤之王挑战赛页面点击"挑战"按钮时出现此错误。

### 错误原因
`interfaces/routes/king_routes.py` 文件中使用了错误的导入语句：
```python
from application import services
```

这个导入方式不正确，因为 `application` 模块本身没有 `services` 属性。正确的导入应该是从 `bootstrap` 模块导入已初始化的 `services` 对象。

## 🔧 修复方案

### 文件：`interfaces/routes/king_routes.py`

**修改前**：
```python
from application import services
```

**修改后**：
```python
from interfaces.web_api.bootstrap import services
```

## 📝 详细说明

### 为什么要这样修改？

1. **bootstrap.py 的作用**
   - `interfaces/web_api/bootstrap.py` 是服务初始化模块
   - 它创建并配置了所有的服务实例（repositories, services等）
   - 所有路由都应该从这里导入 `services` 对象

2. **services 对象包含的内容**
   - `services.player_repo` - 玩家仓库
   - `services.player_beast_repo` - 幻兽仓库
   - `services.beast_pvp_service` - PVP战斗服务
   - 等等...

3. **正确的导入模式**
   ```python
   from interfaces.web_api.bootstrap import services
   
   # 然后可以使用：
   player = services.player_repo.get_by_id(user_id)
   beasts = services.player_beast_repo.get_team_beasts(user_id)
   ```

## ⚠️ 注意事项

### 1. 导入顺序
确保在使用 `services` 之前已经导入：
```python
from interfaces.web_api.bootstrap import services
```

### 2. 循环导入问题
如果遇到循环导入问题，可以在函数内部导入：
```python
def some_function():
    from interfaces.web_api.bootstrap import services
    player = services.player_repo.get_by_id(user_id)
```

### 3. 服务初始化
确保 `bootstrap.py` 中的 `Services` 类已经正确初始化所有需要的服务。

- ❌ 错误：`from application import services`
- ✅ 正确：`from interfaces.web_api.bootstrap import services`

修复后需要重启后端服务才能生效。
