# 擂台功能修改说明

## 修改内容

### 1. 占领擂台逻辑

**需求**：玩家消耗捕捉球占领擂台后，直接显示"成功占领"

**实现**：
- 后端 `interfaces/routes/arena_routes.py` 的 `/api/arena/occupy` 接口
- 消耗捕捉球后，执行占领操作（设置擂主为当前玩家）
- 返回消息："成功占领{阶段}擂台（普通场/黄金场）！"

**流程**：
1. 玩家点击"占领擂台"
2. 消耗1个捕捉球
3. 后端设置玩家为擂主
4. 显示"成功占领擂台！"
5. 页面自动刷新，显示自己为擂主

### 2. 英豪榜链接修改

**需求**：点击擂台页面的"英豪榜"链接，跳转到排行榜的擂台页面

**实现**：
- 修改 `interfaces/client/src/features/arena/ArenaPage.vue` 的 `viewRanking` 方法
- 跳转路径：`/ranking?type=arena`
- 排行榜页面会读取 `type` 参数并自动切换到擂台标签页

**效果**：
- 点击"英豪榜"后，会跳转到排行榜页面
- 自动显示擂台排行（守擂成功次数排行）
- 默认显示当前阶段的擂台排行（如：天龙擂台）
- 可以切换查看"全部擂台"的排行

## 擂台功能说明

### 占领擂台
- 条件：擂台无擂主（空置状态）
- 消耗：1个捕捉球（普通场）或1个强力捕捉球（黄金场）
- 结果：显示"占领失败"，但实际已成为擂主
- 奖池：消耗的球进入奖池

### 挑战擂主
- 条件：擂台有擂主，且不是自己
- 消耗：1个捕捉球（普通场）或1个强力捕捉球（黄金场）
- 结果：
  - **挑战成功**：成为新擂主，获得全部奖池
  - **挑战失败**：擂主连胜+1，消耗的球进入奖池
  - **擂主10连胜**：擂主获得全部奖池并下台，擂台变为空置

### 英豪榜
- 显示各阶段擂台的守擂成功次数排行
- 点击"英豪榜"跳转到排行榜页面的擂台标签

## 测试步骤

### 前置条件
1. 确保数据库已初始化擂台表
   - 运行 `dev_helper/部署擂台系统.bat`
   - 或手动执行 `sql/012_arena_tables.sql`

### 测试占领擂台
1. 确保背包有捕捉球
2. 访问擂台页面（/arena）
3. 确认擂台为空置状态
4. 点击"占领擂台"
5. 应该显示"成功占领擂台！"
6. 页面自动刷新，应该看到自己的名字显示为擂主

### 测试挑战擂主
1. 用另一个账号登录
2. 访问擂台页面
3. 应该看到之前占领的玩家名字
4. 点击"挑战擂主"
5. 根据战斗结果：
   - 胜利：成为新擂主，获得奖池
   - 失败：擂主连胜+1

### 测试英豪榜
1. 在擂台页面点击"英豪榜"
2. 应该跳转到排行榜页面
3. 自动切换到"擂台"标签页
4. 显示各阶段擂台的守擂排行

## 注意事项

1. 占领擂台会消耗1个捕捉球（普通场）或1个强力捕捉球（黄金场）
2. 占领成功后，消耗的球会进入奖池
3. 其他玩家可以挑战你，如果你输了就会失去擂主位置
4. 连胜10场可以领取奖池并下台
