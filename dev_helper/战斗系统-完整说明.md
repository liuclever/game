# 游戏战斗系统完整说明

## 概述

这个游戏已经实现了完整的PVP（玩家对玩家）战斗系统，基于幻兽对战的回合制战斗。

## 战斗系统架构

### 核心组件

1. **战斗引擎** (`domain/services/pvp_battle_engine.py`)
   - 纯领域层模块，不依赖任何框架
   - 负责战斗逻辑计算、伤害计算、先手判定
   - 生成详细的战斗日志

2. **技能系统** (`domain/services/skill_system.py`)
   - 主动技能：必杀、连击、破甲、毒攻、吸血等
   - 被动技能：闪避、反震、反击、幸运等
   - Buff/Debuff效果：减防、减速、中毒等

3. **战斗服务** (Application Layer)
   - 各种战斗场景的业务逻辑封装
   - 战报格式转换
   - 奖励发放

## 战斗流程

### 1. 战前准备
- 获取双方玩家的幻兽队伍（最多6只）
- 将幻兽数据转换为战斗快照（PvpBeast）
- 包含所有属性：气血、攻击、防御、速度、技能等

### 2. 战斗执行
```
while 双方都有存活的幻兽:
    1. 判定先手（速度 > 品质 > 星数 > 资质 > 属性）
    2. 先手方攻击：
       - 判定主动技能触发（必杀/连击/破甲等）
       - 判定被动技能触发（闪避/反震/反击）
       - 计算伤害
       - 扣血
       - 记录战报
    3. 后手方攻击（同上）
    4. 如果有幻兽阵亡，换下一只幻兽上场
```

### 3. 战斗结束
- 判定胜负（一方所有幻兽阵亡）
- 生成完整战报
- 发放奖励

## 先手判定规则

优先级从高到低：

1. **速度值**：考虑Buff后的有效速度，速度高的先手
2. **品质**：神兽 > 精英 > 普通
3. **星数总和**：气血星、攻击星、双防星、速度星之和
4. **资质总和**：五项资质之和
5. **属性总和**：五项属性数值之和
6. **随机**：完全相同则50%概率

## 伤害计算公式

### 破防情况（攻击 > 防御）
```
伤害 = (攻击 - 防御) × 随机(0.069~0.071) × 防御档位倍数
```

**防御档位倍数**：
- 防御 < 1000：3.8倍
- 1000 ≤ 防御 < 2000：2.0倍
- 2000 ≤ 防御 < 3000：1.6倍
- 3000 ≤ 防御 < 4000：1.3倍
- 4000 ≤ 防御 < 5000：1.1倍
- 防御 ≥ 5000：1.0倍

### 未破防情况（攻击 < 防御）
根据攻防差的绝对值：
- |差值| ≤ 1000：250~300
- 1000 < |差值| ≤ 2000：250~300 × 0.7
- 2000 < |差值| ≤ 3000：250~300 × 0.5
- 3000 < |差值| ≤ 4000：250~300 × 0.3
- |差值| > 4000：20~40

**特殊规则**：
- 攻击方等级在20-39级（黄阶/玄阶），未破防伤害额外×0.3
- 最终伤害至少为1

### 主动技能（攻击时触发）

| 技能名 | 效果 | 触发概率 |
|--------|------|----------|
| 必杀 | 伤害×1.5 | 20% |
| 高级必杀 | 伤害×2.0 | 25% |
| 连击 | 连续攻击2次 | 20% |
| 高级连击 | 连续攻击3次 | 25% |
| 破甲 | 降低对方防御30%，持续3回合 | 15% |
| 高级破甲 | 降低对方防御50%，持续3回合 | 20% |
| 毒攻 | 使对方中毒，每回合损失5%最大气血 | 15% |
| 高级毒攻 | 使对方中毒，每回合损失8%最大气血 | 20% |
| 吸血 | 吸取造成伤害的30% | 15% |
| 高级吸血 | 吸取造成伤害的50% | 20% |

### 被动技能（受击时触发）

| 技能名 | 效果 | 触发概率 |
|--------|------|----------|
| 闪避 | 完全躲避攻击 | 15% |
| 高级闪避 | 完全躲避攻击 | 25% |
| 反震 | 反弹30%伤害 | 15% |
| 高级反震 | 反弹50%伤害 | 20% |
| 反击 | 反击30%伤害 | 15% |
| 高级反击 | 反击50%伤害 | 20% |
| 幸运 | 降低对方必杀触发率50% | 被动生效 |
| 抗性增强 | 50%概率免疫中毒 | 被动生效 |

### 技能克制关系

- **偷袭** 免疫反震/反击
- **幸运** 大幅降低对方必杀触发率
- **抗性增强** 有概率免疫中毒

## 已实现的战斗场景

### 1. 镇妖塔战斗
- 文件：`application/services/zhenyao_service.py`
- 玩家挑战镇妖塔各层的守护幻兽
- 胜利可占领该层，获得持续奖励

### 2. 擂台战斗
- 文件：`interfaces/routes/arena_routes.py`
- 玩家挑战擂主
- 胜利成为新擂主，获得奖池奖励

### 3. 通天塔战斗
- 文件：`application/services/tower_service.py`
- 玩家挑战通天塔各层
- 逐层推进，获得奖励

### 4. 战场战斗
- 文件：`application/services/battlefield_service.py`
- 玩家之间的PVP对战
- 胜利获得积分和奖励

### 5. 切磋战斗
- 文件：`interfaces/routes/player_routes.py`
- 玩家之间的友谊赛
- 不消耗资源，只记录战绩

### 6. 召唤之王挑战赛
- 文件：`interfaces/routes/king_routes.py`
- 预选赛和正赛
- 排名赛制，晋级淘汰

### 7. 连胜竞技场
- 文件：`interfaces/routes/arena_streak_routes.py`
- 连胜挑战模式
- 连胜越多奖励越高

### 8. 副本挑战
- 文件：`interfaces/routes/dungeon_routes.py`
- 挑战各地图的副本
- 获得经验和掉落

### 9. 盟战
- 文件：`application/services/alliance_battle_service.py`
- 联盟之间的团队战
- 贡献积分，获得联盟奖励

## 战报系统

### 战报生成流程

1. **战斗引擎生成原始日志**
   - 每次攻击生成一条 `AttackLog`
   - 包含：回合数、攻击方、防守方、伤害、技能名、描述文本

2. **业务层转换格式**
   - 将连续的攻击日志按幻兽对阵关系分组
   - 生成前端需要的JSON格式
   - 包含：战斗序列、每场胜负、详细回合信息

3. **前端展示**
   - 逐回合播放战斗过程
   - 显示气血变化
   - 显示技能效果

## 开发建议

### 新增战斗场景
1. 创建新的路由文件（如 `xxx_routes.py`）
2. 获取双方幻兽数据
3. 调用 `run_pvp_battle` 执行战斗
4. 转换战报格式
5. 保存战报到数据库
6. 发放奖励

### 新增技能
1. 在 `configs/skills.json` 添加配置
2. 在 `domain/services/skill_system.py` 实现效果逻辑
3. 测试触发概率和效果

### 调整平衡性
1. 修改 `pvp_battle_engine.py` 中的伤害公式
2. 调整技能触发概率
3. 修改防御档位倍数
