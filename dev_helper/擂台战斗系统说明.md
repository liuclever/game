# 擂台战斗系统说明

## 概述

擂台挑战使用了完整的PVP战斗引擎，挑战者和擂主的幻兽会进行真实的回合制战斗。

## 战斗流程

### 1. 挑战前检查
- 检查玩家等级（20级以上）
- 检查今日挑战次数（VIP等级决定次数上限）
- 检查捕捉球数量
- 检查擂台是否有擂主

### 2. 消耗资源
- 扣除1个捕捉球（普通场）或1个强力捕捉球（黄金场）
- 增加今日挑战次数
- 捕捉球进入奖池

### 3. 战斗执行

```python
# 获取双方幻兽队伍
challenger_beasts = services.player_beast_repo.get_team_beasts(user_id)
champion_beasts = services.player_beast_repo.get_team_beasts(champion_user_id)

# 转换为战斗快照
attacker_pvp_beasts = services.beast_pvp_service.to_pvp_beasts(challenger_beasts)
defender_pvp_beasts = services.beast_pvp_service.to_pvp_beasts(champion_beasts)

# 创建战斗玩家
attacker_player = PvpPlayer(
    player_id=user_id,
    level=player_level,
    beasts=attacker_pvp_beasts,
    name=player_nickname,
)

defender_player = PvpPlayer(
    player_id=champion_user_id,
    level=champion_level,
    beasts=defender_pvp_beasts,
    name=champion_nickname,
)

# 执行战斗
pvp_result = run_pvp_battle(attacker_player, defender_player, max_log_turns=50)
```

### 4. 战斗结果处理

#### 挑战者胜利
- 成为新擂主
- 获得全部奖池
- 连胜数重置为0
- 奖池清空

#### 挑战者失败
- 擂主连胜+1
- 如果擂主达到10连胜：
  - 擂主获得全部奖池
  - 擂主下台（擂台变为空置）
  - 守擂成功次数+1（用于排行榜）
- 否则：
  - 擂主继续守擂

### 5. 战报保存

战斗结果会保存到 `arena_battle_log` 表，包含：
- 战斗双方信息
- 胜负结果
- 详细战报JSON（每回合的攻击记录）
- 战报ID（用于查看详细战报）

## 战斗特点

### 真实战斗
- 使用完整的PVP战斗引擎（`domain/services/pvp_battle_engine.py`）
- 双方幻兽按顺序1v1对战
- 支持所有技能系统（必杀、连击、闪避、反震等）
- 考虑先手判定、伤害计算、技能触发等

### 战报系统
- 记录每回合的攻击详情
- 包含技能触发、伤害数值、气血变化
- 可以查看详细战报回放

### 兜底机制
如果双方都没有幻兽，则：
- 挑战者自动失败
- 不生成战报
- 正常扣除资源和增加连胜

## 战报查看

### 后端接口
```python
@arena_bp.get("/battle/<int:battle_id>")
def get_arena_battle_detail(battle_id: int):
    """获取擂台战斗详情（用于详细战报页面）"""
    log = ARENA_BATTLE_REPO.get_by_id(battle_id)
    if not log:
        return jsonify({"ok": False, "error": "战报不存在"}), 404
    return jsonify({"ok": True, "battle": log.to_dict()})
```

### 前端页面
- 文件：`interfaces/client/src/features/arena/ArenaBattlePage.vue`
- 路由：`/arena/battle?id={battle_id}`
- 功能：逐回合播放战斗过程

## 擂台动态

### 全服动态
显示所有擂台的最近战斗记录：
```python
@arena_bp.get("/dynamics?type=arena")
```

### 个人动态
显示玩家自己的擂台战斗记录：
```python
@arena_bp.get("/dynamics?type=personal")
```

动态信息包含：
- 时间
- 挑战者/擂主名字
- 战斗结果
- 战报链接（可查看详细战报）

## VIP特权

不同VIP等级有不同的每日挑战次数：

| VIP等级 | 普通场次数 | 黄金场次数 |
|---------|-----------|-----------|
| VIP 0   | 5次       | 10次      |
| VIP 1+  | 根据配置  | 根据配置  |

配置文件：`configs/vip_privileges.json`

## 数据库表

### arena（擂台表）
- 存储各阶段擂台的当前状态
- 擂主信息、连胜数、奖池

### arena_battle_log（战报表）
- 存储所有擂台战斗记录
- 包含详细战报JSON

### arena_stats（守擂统计表）
- 记录玩家达成10连胜的次数
- 用于英豪榜排行

### arena_daily_challenge（每日挑战表）
- 记录玩家每日挑战次数
- 每天重置
