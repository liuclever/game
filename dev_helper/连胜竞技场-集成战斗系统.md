# 连胜竞技场 - 集成真实战斗系统

## ✅ 已完成集成

### 后端集成 (`interfaces/routes/arena_streak_routes.py`)

连胜竞技场的 `/battle` 接口已经集成了真实的PVP战斗引擎：

1. **获取双方幻兽队伍**
   - 使用 `player_beast_repo.get_team_beasts()` 获取出战幻兽
   - 使用 `beast_pvp_service.to_pvp_beasts()` 转换为战斗快照

2. **执行真实战斗**
   - 使用 `run_pvp_battle()` 执行完整的PVP战斗
   - 包含技能系统、先手判定、伤害计算等

3. **生成战报**
   - 从 `pvp_result.logs` 提取真实战斗日志
   - 返回简略版（前4-6条）和详细版（完整战报）

4. **更新连胜记录**
   - 根据真实战斗结果更新连胜次数
   - 胜利：连胜+1
   - 失败：连胜归零

### 前端集成

1. **ArenaStreak.vue**
   - 点击"切磋"触发战斗
   - 将战报数据存储到 sessionStorage
   - 跳转到战报页面

2. **ArenaStreakBattleReport.vue**
   - 从 sessionStorage 读取战报数据
   - 显示战斗结果（胜利/失败）
   - 显示当前连胜次数
   - 显示简略战报（前几条）
   - 可展开查看详细战报（完整战斗过程）

## 🎮 战斗系统特性

### 真实战斗计算
- 基于双方幻兽属性（气血、攻击、防御、速度）
- 完整的技能系统（必杀、连击、破甲、毒攻、闪避、反震、反击等）
- 多级先手规则（速度 > 品质 > 星数 > 资质 > 属性）
- 精确的伤害公式（攻防差 + 浮动系数 + 防御档位倍数）

### 战报生成
- 每次攻击生成一条战报
- 包含技能名称、伤害数值、气血变化
- 支持简略版和详细版切换

## 📝 数据流程

```
用户点击"切磋"
  ↓
前端发送 POST /api/arena-streak/battle
  ↓
后端获取双方幻兽队伍
  ↓
执行 run_pvp_battle() 真实战斗
  ↓
生成战报日志
  ↓
更新连胜记录
  ↓
返回战斗结果 + 战报
  ↓
前端存储到 sessionStorage
  ↓
跳转到战报页面
  ↓
显示战斗结果和战报
```

## 🔧 技术细节

### 后端返回数据结构
```json
{
  "ok": true,
  "victory": true/false,
  "current_streak": 5,
  "message": "胜利！当前连胜5次",
  "attacker_name": "玩家昵称",
  "defender_name": "对手昵称",
  "battle_logs": ["简略战报1", "简略战报2", ...],
  "detail_logs": ["详细战报1", "详细战报2", ...]
}
```

### 前端数据传递
- 使用 `sessionStorage` 在页面间传递战报数据
- 避免 URL 参数过长
- 战报页面读取后立即清除 sessionStorage

## ⚠️ 注意事项

1. **不消耗活力**
   - 连胜竞技场的切磋不消耗活力
   - 虽然代码中有 `energy_cost` 变量，但实际不扣除

2. **战报长度限制**
   - 简略版：前4-6条战报
   - 详细版：最多50条战报（由 `max_log_turns=50` 控制）

3. **连胜机制**
   - 胜利：连胜+1，更新今日最高连胜
   - 失败：连胜归零
   - 连胜记录用于领取奖励
