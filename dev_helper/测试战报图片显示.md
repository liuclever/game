# 测试战报图片显示

## 问题
战报页面的幻兽图片没有根据出战角色匹配显示

## 原因
`_build_spar_battle_data` 函数构建的幻兽信息中缺少 `template_id` 字段

## 修复内容

### 1. 修改 `interfaces/routes/player_routes.py`
在 `_build_spar_battle_data` 函数中，为幻兽信息添加 `template_id` 字段：

```python
attacker_beasts_info.append({
    "name": beast.name,
    "realm": realm_name,
    "exp_gain": 0,
    "template_id": beast.template_id if hasattr(beast, 'template_id') else None,
})
```

### 2. 前端已正确实现
- `KingBattleReportPage.vue` - 召唤之王战报 ✅
- `SparBattleReportPage.vue` - 切磋战报 ✅
- `ArenaStreakBattleReport.vue` - 擂台战报 ✅

所有页面都使用相同的图片加载逻辑：
```javascript
const beastImageModules = import.meta.glob('@/assets/images/image*.jpeg', { eager: true })
const getBeastImage = (templateId) => {
  if (!templateId) return ''
  const key = `/src/assets/images/image${templateId}.jpeg`
  const module = beastImageModules[key]
  return module?.default || ''
}
```

## 测试步骤

1. **清除浏览器缓存**
   - 按 Ctrl+Shift+Delete
   - 或者使用无痕模式测试

2. **重启后端服务**
   ```bash
   # 停止当前服务（Ctrl+C）
   # 重新启动
   cd dev_helper
   启动后端.bat
   ```

3. **测试切磋战报**
   - 进入竞技主页
   - 点击切磋
   - 查看战报，确认幻兽图片正确显示

4. **测试擂台战报**
   - 进入连胜竞技场
   - 开始战斗
   - 查看战报，确认幻兽图片正确显示

5. **测试召唤之王战报**
   - 进入召唤之王
   - 挑战对手
   - 查看战报，确认幻兽图片正确显示

## 预期结果

所有战报页面应该显示：
- 幻兽图片（48x48像素）
- 幻兽名称在图片下方
- 左右两排幻兽，中间显示"PK"
- 点击每一战可以展开详细回合信息

## 如果仍然看不到图片

1. 检查幻兽是否有 `template_id`
   ```sql
   SELECT id, name, template_id FROM player_beasts WHERE user_id = 1;
   ```

2. 检查图片文件是否存在
   ```
   interfaces/client/src/assets/images/image1.jpeg
   interfaces/client/src/assets/images/image2.jpeg
   ...
   ```

3. 检查浏览器控制台是否有错误信息

4. 检查后端返回的数据结构
   - 打开浏览器开发者工具
   - 查看 Network 标签
   - 找到战报API请求
   - 查看返回的 JSON 数据中是否包含 `template_id`
