## 💡 学习笔记

### 项目架构理解
- **domain/**：核心游戏规则层
  - 战斗计算、升级逻辑、掉落规则等
  - 不依赖任何框架和数据库
  
- **application/**：业务服务层
  - 串联各种规则，形成完整的业务流程
  - 例如：签到服务、战斗服务、地图服务
  
- **interfaces/**：接口层
  - 前端 Vue 项目
  - 后端 Flask API 路由
  
- **infrastructure/**：基础设施层
  - 数据库访问实现
  - 配置文件加载
  - 日志、缓存等
  
- **configs/**：配置数据层
  - 游戏数值配置（JSON）
  - 策划可以直接修改，不需要改代码

### 技术栈
- 后端：Flask + PyMySQL
- 前端：Vue 3 + Vite
- 数据库：MySQL 8.0
- 架构：领域驱动设计（DDD）
