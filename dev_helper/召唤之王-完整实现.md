# 召唤之王挑战赛 - 完整实现文档

## 功能概述

召唤之王挑战赛是一个完整的跨服竞技系统，包含预选赛和正赛两个阶段，所有功能已全部实现。

## 已实现功能 ✅

### 1. 赛区分配系统 ✅
- 建号时自动分配赛区（1区或2区）
- 保证两个赛区人数一致（差距不超过1人）
- 新玩家排在赛区最后

### 2. 预选赛系统 ✅
- 报名系统（星期一可报名）
- 每日15次免费挑战
- 挑战冷却（60秒）
- 并发控制（数据库行锁）
- 只能挑战排名更高的玩家
- 显示前5名可挑战对手
- 胜利交换排名，失败排名不变
- 预选赛奖励（胜利20000，失败2000）
- 连胜统计

### 3. 正赛系统 ✅
- 周四23:59自动晋级（各赛区前16名）
- 32人随机匹配
- 16强→8强→4强→2强→冠军
- 轮空处理（人数不足时）
- 各阶段奖励自动发放
- 冠军获得"召唤之王"称号

### 4. 定时任务 ✅
- 周一 00:00 - 重置报名状态
- 周四 23:59 - 晋级正赛
- 周五 12:00 - 32强赛
- 周五 13:00 - 16强赛
- 周五 14:00 - 8强赛
- 周五 15:00 - 4强赛
- 周五 16:00 - 决赛

### 5. 前端功能 ✅
- 报名按钮（仅星期一显示）
- 冷却倒计时实时显示
- 挑战按钮状态管理
- 英豪榜（可切换赛区）
- 正赛信息查询API

## 系统架构

### 数据库表

#### king_challenge_rank（排名表）
```sql
CREATE TABLE king_challenge_rank (
    user_id INT PRIMARY KEY,
    area_index INT NOT NULL DEFAULT 1,           -- 赛区编号
    rank_position INT NOT NULL,                  -- 排名
    win_streak INT NOT NULL DEFAULT 0,           -- 连胜场次
    total_wins INT NOT NULL DEFAULT 0,           -- 总胜场
    total_losses INT NOT NULL DEFAULT 0,         -- 总负场
    today_challenges INT NOT NULL DEFAULT 0,     -- 今日挑战次数
    last_challenge_date DATE DEFAULT NULL,       -- 上次挑战日期
    last_challenge_time DATETIME DEFAULT NULL,   -- 最后挑战时间（冷却）
    is_registered TINYINT(1) NOT NULL DEFAULT 0, -- 是否已报名本周
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_area_rank (area_index, rank_position)
);
```

#### king_final_stage（正赛阶段表）
```sql
CREATE TABLE king_final_stage (
    id INT AUTO_INCREMENT PRIMARY KEY,
    season INT NOT NULL,                         -- 赛季编号
    user_id INT NOT NULL,                        -- 玩家ID
    stage VARCHAR(20) NOT NULL,                  -- 阶段：32/16/8/4/2/champion
    match_id INT DEFAULT NULL,                   -- 对战ID
    opponent_id INT DEFAULT NULL,                -- 对手ID
    is_bye TINYINT(1) NOT NULL DEFAULT 0,       -- 是否轮空
    is_winner TINYINT(1) DEFAULT NULL,          -- 是否胜利
    battle_time DATETIME DEFAULT NULL,           -- 战斗时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_season_stage (season, stage),
    INDEX idx_user (user_id)
);
```

#### king_season_config（赛季配置表）
```sql
CREATE TABLE king_season_config (
    season INT PRIMARY KEY,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'registration',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### king_reward_claimed（奖励领取记录）
```sql
CREATE TABLE king_reward_claimed (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    season INT NOT NULL,
    reward_tier VARCHAR(20),
    claimed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_season (user_id, season)
);
```

## 核心特性

### 1. 并发安全
- 使用数据库事务
- 使用行锁（`SELECT ... FOR UPDATE`）
- 防止排名冲突
- 防止同时挑战同一个人

### 2. 轮空处理
- 人数不是2的幂次方时，部分选手轮空
- 轮空选手直接晋级下一轮
- 轮空选手也能获得当前轮次奖励

### 3. 自动化
- 定时任务自动执行
- 奖励自动发放
- 报名状态自动重置
- 冠军称号自动更新

## 注意事项

1. **赛区分配**：只在首次创建排名记录时分配，之后不会改变
2. **冷却时间**：60秒，从挑战完成时开始计算
3. **报名限制**：只能在星期一报名
4. **并发控制**：使用行锁，如果两个人同时挑战同一个目标，后者会等待
5. **事务回滚**：任何错误都会回滚事务，保证数据一致性
6. **定时任务**：自动启动，无需手动触发
7. **奖励发放**：正赛奖励在晋级时自动发放，无需手动领取
8. **冠军称号**：每周只有一个冠军，称号会自动更新
