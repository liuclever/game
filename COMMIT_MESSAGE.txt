实现镇妖动态功能完善

问题：
- 镇妖界面的全服动态和个人动态没有正确显示
- 占领操作没有生成动态记录
- 动态文本格式不够清晰友好

修复内容：

1. 添加占领动态记录
   - occupy_floor() 方法中添加动态记录
   - 占领成功后保存到 zhenyao_battle_log 表
   - defender_id = 0 表示占领（无防守方）
   - 记录类型标记为 "occupy"

2. 优化动态文本格式化
   - _format_dynamic_text() 方法重写
   - 区分占领和挑战两种类型
   - 占领: 【玩家】成功占领第X层聚魂阵
   - 挑战成功: 【挑战者】挑战【防守者】占领的第X层聚魂阵，挑战成功！
   - 挑战失败: 【挑战者】挑战【防守者】占领的第X层聚魂阵，挑战失败

3. 前端动态显示优化
   - 使用后端返回的 text 字段
   - 移除硬编码的动态文本拼接
   - 简化显示逻辑
   - 时间显示优化

4. 测试验证
   - test_zhenyao_dynamics.py
   - 测试全服动态和个人动态
   - 验证动态文本格式
   - 测试占领动态生成

动态类型：
- 占领动态: 玩家占领无人占领的层
- 挑战成功动态: 玩家挑战成功并抢夺层
- 挑战失败动态: 玩家挑战失败

动态文本示例：
- 【张三】成功占领第31层聚魂阵
- 【李四】挑战【张三】占领的第31层聚魂阵，挑战成功！
- 【王五】挑战【赵六】占领的第40层聚魂阵，挑战失败

技术细节：
- 占领动态: defender_id = 0, defender_name = ""
- 挑战动态: defender_id > 0, defender_name 有值
- 动态按时间倒序排列
- 支持全服动态和个人动态筛选

前端改进：
- 直接使用后端返回的 text 字段
- 移除复杂的文本拼接逻辑
- 时间显示更简洁
- 成功动态显示绿色
