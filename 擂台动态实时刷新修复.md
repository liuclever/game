# 擂台动态实时刷新修复

## 问题描述
玩家在擂台挑战或占领擂台后，擂台动态列表没有实时刷新，需要手动刷新页面才能看到最新的战斗记录。

## 问题原因
在 `ArenaPage.vue` 中，`challenge()` 和 `occupyArena()` 函数在操作成功后只调用了 `loadArenaInfo()` 来刷新擂台信息（擂主、奖池等），但没有调用 `loadDynamics()` 来刷新擂台动态列表。

## 修复方案
在挑战和占领操作成功后，同时调用 `loadDynamics()` 刷新动态列表。

### 修改文件
`interfaces/client/src/features/arena/ArenaPage.vue`

### 修改内容

#### 1. 占领擂台函数
```javascript
// 修改前
if (res.data.ok) {
  resultMessage.value = res.data.message
  loadArenaInfo()
}

// 修改后
if (res.data.ok) {
  resultMessage.value = res.data.message
  loadArenaInfo()
  loadDynamics() // 刷新擂台动态
}
```

#### 2. 挑战擂主函数
```javascript
// 修改前
if (res.data.ok) {
  resultMessage.value = res.data.message
  loadArenaInfo()
}

// 修改后
if (res.data.ok) {
  resultMessage.value = res.data.message
  loadArenaInfo()
  loadDynamics() // 刷新擂台动态
}
```

## 修复效果
- ✅ 占领擂台后，擂台动态立即显示占领记录
- ✅ 挑战擂主后，擂台动态立即显示挑战结果
- ✅ 无需手动刷新页面即可看到最新动态
- ✅ 提升用户体验，实时反馈战斗结果

## 测试建议
1. 玩家A占领空擂台，检查动态列表是否立即显示占领记录
2. 玩家B挑战玩家A，检查动态列表是否立即显示挑战记录
3. 切换"擂台动态"和"个人动态"标签，验证数据正确性
4. 在普通场和黄金场之间切换，验证动态刷新正常

## 相关功能
- 擂台信息刷新：`loadArenaInfo()` - 刷新擂主、奖池、挑战次数等
- 擂台动态刷新：`loadDynamics()` - 刷新战斗记录列表
- 动态类型切换：支持"擂台动态"（所有玩家）和"个人动态"（当前玩家）
