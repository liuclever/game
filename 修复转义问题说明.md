# 修复转义问题说明

## 问题描述

在批量移除alert时，Python脚本错误地转义了单引号，导致：
```javascript
// 错误的代码
import { useMessage } from \'@/composables/useMessage\'

// 正确的代码
import { useMessage } from '@/composables/useMessage'
```

## 错误信息

```
[vue/compiler-sfc] Expecting Unicode escape sequence \uXXXX.
```

## 原因分析

Python的 `re.sub()` 函数在替换时，将单引号 `'` 错误地转义为 `\'`。

## 解决方案

创建了修复脚本 `fix_escaped_quotes.py`，批量将所有 `\'` 替换回 `'`。
